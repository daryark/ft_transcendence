//***app.js, server.js, socket.js
//->app.js
- creates express instance
- defines middlewares (cors, json)
- defines routes (/health, /api/...)
- NO network, NO sockets
//->sever.js
- creates HTTP server from app
- attaches Socket.IO
- starts listening on PORT
//->socket.js
- handles socket connections
- routes socket events
- connects handlers (game/chat)

//***Express.js***
//-> Framework that handles http routing
const express = require('express');
//-> http - core node module to create http server
const http = require('http');

//->app object
const app = express();
app object - represents the whole Express app
it is used to define routes and middleware, and handling HTTP requests/responses

#1
//->Middlewares: app.use() - functions using which you have access to the req & res, next!
and next()!!! middleware function in the application's req-res cycle.
App object is used to apply middleware to your applicaiton.
#2
//->CORS middleware. Simple Usage (Enable All CORS Requests)
const cors = require('cors');
app.use(cors());
#3
//-> /health
app.get('/health', () => {..})
health path is being used by load-balancers, docker, kubernetes
to ping, if server is running or there's a need to restart container and/or connections.

//***Socket.IO***

#4
//->Socket properties/attributes
socket.id - unique connection id (changes randomly during the same connection)
socket.rooms - rooms this socket joined
socket.data - your custom storage,data is a built-in plain! object provided by Socket.IO, you can custom fileds inside to attach info to a specific socket connection
#5
//->socket.on(eventName, listener);
    socket.on("details", (...args) => { });
Adds the listener function to the end of the listeners array for the event named eventName
//->eventName on socket.on();
Socket.IO event names are just strings, but you design them as:
<domain>:<action>   ("quickplay:join"/"league:join"/"game:state"/"room:join"...)

//***Jest***
//->jest.fn()
fake function that imitates something that doesnt' exist in test context, but need to be (to not fail).
Ex: const socket = { id: 's1', join: jest.fn() }
socket is not real (on real socket there's a method socket.join()), so we immitate that it exists
and can check, if this fake function was called in the code is we want it.
